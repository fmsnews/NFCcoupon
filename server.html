Server Website

The First Thing

นอกจากการใช้แอพพลิเคชั่น WingBonus แล้วยังมีเว็บไซต์ที่พัฒนาโดยการใช้ content management systems และใช้ภาษา HTML5 , CSS3 และ JavaScript ในการพัฒนา เว็บไซต์ WingBonusจะเป็นเรื่องของการโฆษณาและการจัดการm-couponsบริษัทผู้ใช้(แบรนด์)สินค้าและอื่น ๆ และข้อมูลเกี่ยวกับคูปองที่จะแสดงให้กับผู้ใช้ตามที่มีการตั้งค่าของผู้ใช้ รูปที่ 1.9 (a) หน้าหลักสำหรับการเข้าถึงบัญชีและการจัดการรายการคูปองล่าสุดที่มีการเสนอโดยระบบ

The Third Thing

โดยถ้าคลิกที่คูปองแต่ละอัน ระบบจะแสดงรายละเอียดข้อมูลดังกล่าว เช่น เงื่อนไขและการระบุตำแหน่งผ่านทาง GoogleMapตามที่แสดงในรูปที่ 1.10 (b)

The Third Thing

ผู้ใช้สามารถเลือก(ทำเครื่องหมาย) คูปองจากเว็บไซต์เพื่อที่จะเตรียมใช้งานในภายหลังและเอาออกจากฐานข้อมูลของมือถือ นอกจากนี้ ผู้ใช้สามารถเข้าถึงประวัติการเคลื่อนไหวคูปองด้วยข้อมูลที่ได้ในกระบวนการซิงโครไนส์ ดังแสดงในรูปที่ 1.11 (c)

The Third Thing

รายงานเหล่านี้ได้รวบรวมข้อมูลเกี่ยวกับข้อมูลทั้งหมดที่เกี่ยวข้องกับการประมวลผลคูปองอย่างเช่นข้อมูลเกี่ยวกับ แบรนด์ การแลกเปลี่ยนคะแนน การจัดเตรียมการใช้งาน การถ่ายโอนและการแลกเปลี่ยนคูปอง และอื่นๆ เว็บไซต์ wingbonus ไม่ได้เป็นการมุ่งเน้นที่ผู้ใช้เพียงอย่างเดี่ยว แต่นี้ยังมี gatewayการเข้าถึงทางเดียวสำหรับผู้ดูแลระบบอนุญาตให้แทรกแซงและแก้ไขข้อมูลทุกประเภทที่มีอยู่ในระบบจากลูกค้าองค์กรกับคูปองที่ออกในเวลาใดๆ เช่นตัวอย่างของรูปแบบสำหรับการแนะนำคูปองใหม่ในระบบ แสดงในรูปที่ 1.12 (d)

Synchronization process

Wingbonusประกอบด้วยโปรโตคอลที่พัฒนาด้วย Java ที่สร้างขึ้นด้วยตนเอง โดยการใช้ โปรโตคอลSecure Sockets Layer (SSL) สำหรับกระบวนการ synchronization ระหว่างโทรศัพท์มือถือและฝั่งเซิร์ฟเวอร์จะเป็นการสื่อสารโดยตรงจากผู้ใช้โทรศัพท์มือถือไปยังฐานข้อมูลเซิร์ฟเวอร์อย่างไรก็ตาม, Android ใช้ SQLite และก็ไม่ได้มีการเข้าถึงฐานข้อมูล SQL ภายนอกดังนั้นจะต้องมีการ synchronization ระหว่างการพัฒนาฐานข้อมูลมือถือและเซิร์ฟเวอร์เราเลือกที่จะใช้ที่ระดับต่ำของการแทนของความเป็นไปได้อื่น ๆ เช่น web servicesตัวอย่างเช่นตัวเลือกสำหรับการใช้โปรโตคอลการสื่อสารที่ใช้ เช่น โปรโตคอล SOAPหรือส่งไฟล์ XML ที่จะแยกโปรแกรมมือถือสำหรับแยกข้อมูลที่ร้องขอในภายหลังถึงแม้ว่าในบางครั้งการตรวจสอบความถี่การถ่ายโอนข้อมูลระหว่างโทรศัพท์มือถือ และการประยุกต์ใช้เซิร์ฟเวอร์อาจจะต่ำ ( จำนวนไม่กี่ชิ้นของคูปอง )โดยปกติแล้วกระบวนการsynchronizationจะต้องมีการแลกเปลี่ยนของการตั้งค่าใหม่ดาวน์โหลดคูปองใหม่อัพโหลดคูปองมือถือและอื่นๆ ในกรณีใด ๆ เวลาที่ร้องขอจะใช้เวลาเพียงเล็กน้อย ดังนั้นการสื่อสารJava multithreading ซ็อกเก็ตเซิร์ฟเวอร์ถูกใช้ให้ควบคุมผ่านการรักษาความปลอดภัยขั้นสูงและความสมบูรณ์ของข้อมูลการพัฒนาโปรโตคอลsynchronization(การประสาน) แตกต่างกันหลายชนิดของmessages สำหรับข้อมูลแต่ละประเภทของข้อมูลที่แลกเปลี่ยนความหมายของข้อความเหล่านี้คือ " Login " เพื่อระบุผู้ใช้กับเซิร์ฟเวอร์“get latest coupons “(รับคูปองล่าสุด) สำหรับการดาวน์โหลดคูปองล่าสุดจะแสดงและเลือกจากโมบายแอพพลิเคชั่นโดยผู้ใช้“sync marked”(ซิงค์ทำเครื่องหมาย) เพื่อดาวน์โหลดคูปองที่ทำเครื่องหมายไว้โดยไม่ต้องพิจารณาที่พวกเขาได้เลือก“sync exchanged”(ซิงค์แลกเปลี่ยน) สำหรับการอัพโหลดไปยังเซิร์ฟเวอร์ WingBonusข้อมูลของการใช้งานและ “syncdownloaded”(ซิงค์ดาวน์โหลด) เพื่อส่งข้อมูลของคูปองที่ดาวน์โหลด
